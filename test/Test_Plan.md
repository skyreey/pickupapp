## 2. 测试策略

### 2.1 测试类型与比例

| 类型 | 占比 | 说明 |
|------|------|------|
| 手动探索性测试 | 40% | 真机操作，覆盖真实使用场景 |
| 基于用例的功能测试 | 30% | 按Test_Cases.md逐条验证 |
| 单元测试 | 20% | Jest运行 `__tests__/` 已有6个文件 |
| 兼容性测试 | 10% | ADB注入 + 不同厂商权限流程 |

### 2.2 测试重点

1. **SMS解析准确率** — 用adb注入20+条不同类型的取件短信，验证准确率
2. **去重逻辑** — 同一取件码重复注入，验证不创建重复包裹
3. **通知监听** — 安装淘宝/京东等App，触发真实通知验证
4. **边界条件** — 空数据、大量数据(100+)、异常输入
5. **权限流程** — 未授权/拒绝/系统关闭权限后的App行为

### 2.3 不覆盖范围

- iOS平台（无iOS版本）
- 性能压测（非本次重点）
- 安全渗透测试（独立进行）

## 4. 风险与退出标准

### 4.1 风险

| 风险 | 应对 |
|------|------|
| OCR真机识别率低 | 如<70%，v1.0隐藏OCR Pro入口 |
| 通知监听厂商兼容 | 品牌引导已做，收集具体机型反馈 |
| 物流API覆盖率不足 | 与第三方API供应商确认覆盖范围 |

### 4.2 退出标准

| 级别 | 标准 |
|------|------|
| ✅ 可发布 | P0全部通过 + P1通过率>90% + 无Crash |
| ⚠️ 有条件发布 | P0全部通过 + P1通过率>80% + Crash率<1% |
| ❌ 不可发布 | 存在P0未通过 或 Crash率>1% |
