# 取件通 (PickupApp) — 用户反馈收集方案

> v1.0 上线即启用

---

## 1. 反馈收集渠道

### 1.1 应用内反馈（v1.0内置）

在设置页添加"意见反馈"入口，通过以下方式：

| 方式 | 实现 | 成本 |
|------|------|------|
| 邮件反馈 | 调用系统邮件客户端，发送到 pickupapp.feedback@gmail.com | 零成本 |
| 在线表单 | 设置页内嵌WebView打开腾讯问卷/Google Form | 零成本 |

### 1.2 应用商店渠道

| 渠道 | 说明 |
|------|------|
| 酷安评论区 | 国内Android用户主要反馈来源，需每日巡查 |
| GitHub Issues | 技术用户首选，每个Release附带Issue模板 |

### 1.3 模板

**GitHub Issue Bug模板**：
```
### 问题描述
[一句话描述]

### 复现步骤
1.
2.
3.

### 预期行为
[应该发生什么]

### 实际行为
[实际发生了什么]

### 设备信息
- 手机型号：
- Android版本：
- App版本：
```

**GitHub Issue 功能请求模板**：
```
### 功能描述
[一句话描述想要的功能]

### 使用场景
[什么情况下你会用到这个功能]

### 期望的体验
[描述你认为理想的使用方式]
```

## 2. 反馈分析流程

### 2.1 收集周期

| 周期 | 操作 |
|------|------|
| 每日 | 查看酷安评论区/评分 |
| 每周 | 汇总本周反馈，分类归档 |
| 每月 | 使用Feedback Analyzer Agent生成分析报告 |

### 2.2 反馈分类标准

| 类型 | 标签 | 示例 |
|------|------|------|
| Bug报告 | `bug` | "华为手机收不到取件通知" |
| 功能请求 | `feature` | "希望支持中通快递" |
| 体验反馈 | `ux` | "取件码字体可以再大一些" |
| 性能问题 | `perf` | "首页加载有点慢" |
| 兼容问题 | `compat` | "小米14无法打开OCR" |

### 2.3 处理优先级

| 严重度 | 标签 | 响应时间 | 示例 |
|--------|------|----------|------|
| P0-阻断 | `critical` | 24h内修复 | App崩溃/数据丢失 |
| P1-严重 | `high` | 72h内修复 | 核心功能异常 |
| P2-一般 | `medium` | 下个版本修复 | 体验问题 |
| P3-建议 | `low` | 按需排期 | 新增功能请求 |
