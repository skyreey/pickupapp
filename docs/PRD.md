## 2. 产品概述

### 2.1 产品名称与定位

**取件通** — 一款 Android 快递包裹自动追踪助手。

一句话定位：**不用手动输入，自动识别所有取件信息，一站式管理你的快递。**

### 2.2 产品愿景

让每个网购用户打开一个App就能看到所有待取的快递，不再翻短信、不再漏取件。

### 2.3 价值主张 (USP)

- **零手动** — 授予权限后自动扫描SMS和通知，用户什么都不用做
- **全覆盖** — SMS + 购物通知 + OCR截图 + 手动录入，四个入口无死角
- **轻量隐私** — 24MB安装包，所有解析在本地完成，数据不上传

### 2.4 目标平台列表

| 平台 | 优先级 | 状态 |
|------|--------|------|
| Android (原生) | P0 | ✅ v1.0 |
| iOS | P2 | ⚪ 待iOS开放SMS权限 |
| Web管理后台 | P2 | ⚪ 待规划 |

### 2.5 商业模式概述

- 免费增值：基础功能全免费，Pro功能一次性买断 ¥12-18
- Pro功能：OCR截图识别、数据导出、桌面挂件、深色主题
- 详见 BRD.md 第4章

## 3. 用户研究

### 3.1 目标用户画像（详见 BRD.md 第3章）

**画像A：高频网购打工人**（22-35岁，月均10-30单）
**画像B：代取家人快递的管理者**（30-50岁，家庭月均20-50单）

### 3.2 核心使用场景

| 场景 | 描述 | 频率 |
|------|------|------|
| 收到取件码短信 | 菜鸟/驿站发来取件码 → App自动识别 → 添加到首页列表 | 每天2-5次 |
| 查看待取件列表 | 打开App首页，一眼看到所有待取包裹 | 每天1-2次 |
| 去驿站取件 | 点击包裹→看到取件码大屏→复制/导航→取件后标记完成 | 每天1-3次 |
| 查看物流 | 待收货的商品到哪了 → 详情页查看物流时间线 | 每周2-3次 |
| 手动录入 | 朋友转发取件码截图 → OCR识别或粘贴文本 | 每周1-2次 |

### 4.2 核心功能详述（用户故事格式）

#### F1: SMS自动识别取件码

> 作为 **网购用户**，我想要 **App自动识别快递短信中的取件码**，以便 **不用手动翻短信就能知道所有待取的包裹**。

| 项目 | 内容 |
|------|------|
| 用户价值 | 核心价值，减少90%的取件管理时间 |
| 业务逻辑 | 1. 收到SMS → SmsReceiver → JS解析 → 匹配规则 → 提取取件码/公司/站点 → 存入SQLite；2. 启动时扫描收件箱历史（可配7/30/90天）→ 去重 → 入库 |
| 交互要求 | 解析成功后发送本地通知 |
| 数据需求 | SMS正文、发件人号码、接收时间 |
| 验收标准 | 1. 菜鸟/顺丰/京东等13+快递公司取件码识别准确率 >95%；2. 重复短信自动合并不创建新包裹；3. 已取件短信自动标记 |

#### F2: 购物通知实时监听

> 作为 **多平台网购用户**，我想要 **App自动监听购物App的通知**，以便 **即使短信被拦截也能跟踪包裹状态**。

| 项目 | 内容 |
|------|------|
| 用户价值 | 补充SMS之外的第二数据源，增加覆盖率 |
| 业务逻辑 | NotificationListenerService监听14个购物App → 过滤物流相关通知 → 解析商品名/单号/状态 → 与已有包裹关联 |
| 交互要求 | 每个App可独立开关 |
| 数据需求 | 通知包名、标题、内容、时间 |
| 验收标准 | 1. 启动后自动获取当前活跃通知快照；2. 新通知<3s内解析完成；3. 不重复创建已有包裹 |

#### F3: 包裹列表与详情

> 作为 **多快递用户**，我想要 **一个聚合列表看到所有包裹状态**，以便 **快速决定今天要不要去驿站**。

| 项目 | 内容 |
|------|------|
| 用户价值 | 日常使用的核心界面 |
| 业务逻辑 | 按status分组显示（待取件/进行中/已取件）→ 支持搜索 + 长按多选 → 批量标记取件/删除 |
| 交互要求 | 取件码大号字体、一键复制、一键导航、一键拨号、左右滑动标记取件 |
| 数据需求 | 所有包裹字段 |
| 验收标准 | 1. 列表滚动流畅（60fps）；2. 筛选切换<200ms；3. 搜索实时过滤 |

#### F4: OCR截图识别

> 作为 **收到朋友转发取件码截图**的用户，我想要 **拍照/选图自动识别取件信息**，以便 **不用手动录入取件码**。

| 项目 | 内容 |
|------|------|
| 用户价值 | Pro功能，解决非标准格式的取件信息录入 |
| 业务逻辑 | 调用ML Kit中文OCR → 识别文字 → ocr-parser二次解析提取单号/公司/状态 |
| 交互要求 | 支持多图批量选择 |
| 数据需求 | 图片URI列表 |
| 验收标准 | 1. 单张识别<3s；2. 中文快递单号识别准确率>85%；3. 支持淘宝/京东/拼多多订单截图 |

#### F5: 手动录入

> 作为 **有特殊录入需求的用户**，我想要 **粘贴任意格式的取件信息**，以便 **App自动帮我解析**。

| 项目 | 内容 |
|------|------|
| 用户价值 | 兜底入口，确保覆盖率 |
| 业务逻辑 | 双模式：粘贴全文自动解析 / 手动填入各字段 |
| 交互要求 | Tab切换两种模式 |
| 验收标准 | 1. 粘贴模式支持任意格式文本解析；2. 手动模式所有字段可填 |

#### F6: 物流追踪

> 作为 **关心包裹到哪了的用户**，我想要 **在App内看到物流轨迹**，以便 **不用各平台来回查**。

| 项目 | 内容 |
|------|------|
| 用户价值 | 减少跨App查询 |
| 业务逻辑 | 定时/手动调用物流API → 解析轨迹 → 自动检测签收 → markAsPickedUp |
| 交互要求 | 时间线UI展示物流节点 |
| 验收标准 | 1. 支持主流快递公司物流查询；2. 自动签收检测延迟<2h；3. 定时刷新间隔可配 |

## 5. 用户流程与交互设计

### 5.1 核心用户旅程

```
首次打开App
  → 引导页（品牌分级权限说明）
  → 授予SMS权限
  → 启用通知监听
  → 自动扫描历史短信（7天默认）
  → 显示包裹列表

日常使用
  → 打开App → 首页查看待取件列表
  → 点击包裹 → 详情页（取件码+站点+物流）
  → 取件后 → 左滑标记取件 / 详情页"已取件"按钮

收到新取件短信
  → 通知栏弹出"新包裹到站"通知
  → 点击通知 → App详情页
```

### 5.2 关键状态流转

```
包裹生命周期：
  pending → shipped → in_transit → arrived → stored → picked_up

取件码关联流程：
  购物通知(有单号无取件码) → 创建pending包裹
  ↓
  收到SMS取件码 → findMatchingPackage → 更新取件码+状态→stored
```

### 5.3 对UI/UX的设计要求

- **取件码为第一视觉重心**：详情页取件码使用大号加粗字体（32px+），独立色块背景
- **状态可视化**：不同包裹状态使用颜色区分（待取件=橙，进行中=蓝，已取件=灰）
- **操作零步长**：首页卡片底部直接放"标记取件"按钮，无需进入详情页
- **导航快捷**：取件码和电话均可一键操作（点击复制/拨号/导航）

## 7. 技术架构考量

### 7.1 技术栈

| 层 | 技术 |
|----|------|
| 框架 | Expo 55 (Bare Workflow) |
| 语言 | TypeScript + Kotlin |
| UI | React Native 0.83 + expo-router |
| 数据库 | expo-sqlite |
| OCR | Google ML Kit Chinese |
| 后台任务 | expo-task-manager + expo-notifications |

### 7.2 数据模型（核心）

```
packages 表
├── id: TEXT PRIMARY KEY (UUID)
├── trackingNumber: TEXT (快递单号)
├── carrier: TEXT (快递公司代号)
├── carrierName: TEXT (快递公司名称)
├── orderSource: TEXT (订单来源App)
├── productName: TEXT (商品名称)
├── pickupCode: TEXT (取件码) ← 核心字段
├── pickupAddress: TEXT (取件地址)
├── pickupPointName: TEXT (站点名称)
├── pickupPointPhone: TEXT (站点电话)
├── currentStatus: TEXT (状态)
├── source: TEXT (sms/notification/manual/ocr)
├── createdAt: INTEGER (创建时间)
├── pickedUpAt: INTEGER (取件时间)
└── notes: TEXT (备注)
```

### 7.3 原生模块依赖

| 模块 | 功能 | 权限 |
|------|------|------|
| expo-sms-reader | 短信读取+监听 | READ_SMS / RECEIVE_SMS |
| expo-notification-reader | 通知监听 | BIND_NOTIFICATION_LISTENER_SERVICE |
| expo-ocr-reader | ML Kit OCR | CAMERA（拍照）/ READ_EXTERNAL_STORAGE |
| expo-app-scanner | 已安装App检测 | QUERY_ALL_PACKAGES |
| expo-gallery-launcher | 多图选择 | READ_EXTERNAL_STORAGE |
| expo-pickup-foreground | 前台服务保活 | FOREGROUND_SERVICE |
