## 2. 产品概述

### 2.1 产品名称与定位
取件通是一款面向中国用户的 Android 快递包裹自动追踪助手。
定位为“零操作的个人快递管家”：用户只需授权 SMS 和通知权限，App 自动识别所有取件信息并统一管理。

### 2.2 产品愿景与使命
“让每个中国人的快递取件体验变得简单。”
- 消除手动复制取件码的痛苦
- 一站式管理所有快递入库包裹
- 让代取件变得无缝——家人之间的包裹分享和代取流转不再靠截图+微信

### 2.3 价值主张 (USP)

| 价值点 | 说明 |
|------|------|
| 零手动输入 | SMS自动扫描＋通知栏监控 |
| 全平台覆盖 | 13+快递公司短信解析＋14个购物App通知监控 |
| 家庭代取 | 微信分享→自动识别→弹窗接受/拒绝，一键导入 |
| 数据本地优先 | 100%本地存储，不上传服务器 |
| OCR截图 | ML Kit中文OCR，批量提取单号/公司/状态 |

### 2.4 目标平台

| 平台 | 优先级 | 状态 |
|------|------|------|
| Android (API 26+) | P0 | v1.0已发布 |
| iOS | P2 | 待开发 |

### 2.5 核心假设
1. 中国用户每天平均收到2-3条快递取件短信，但手动管理超过5个包裹时体验急剧下降
2. 家庭成员代取是高频场景，但微信转发取件码+地址的方式低效且易出错
3. 用户对快递数据隐私敏感，不愿意上传到第三方服务器

### 2.6 商业模式

|------|------|------|

## 3. 用户研究

### 3.1 核心用户画像
- 年龄: 18-55岁, 核心28-40岁
- 地域: 一二三四线城市, 以三四线城市为主
- 行为: 每月网购5-30次, 使用多平台, 80%包裹通过驿站取件
- 核心痛点: 取件码分散、找码麻烦、代取流程低效、不确定包裹状态

### 3.2 关键场景

**场景1: 个人取件 (核心)**
作为淘宝常客，我每天收到多条菜鸟取件短信。打开App就能看到所有包裹和取件码，而不是在短信收件箱里一条条翻。

**场景2: 家庭代取 (高频, v2.0重点)**
老婆收到取件码但今天不回家，发微信给我代取。我打开取件通就自动弹窗提示，一键接受并导入。

**场景3: 多包裹管理**
双11期间同时有10多个包裹在路上，我希望知道哪些到了、哪些没到、哪些已经取了。

## 5. 功能需求

### 5.1 功能架构

`
数据采集层:  SMS扫描 | 通知监控 | OCR截图 | 手动录入 | 代取分享
     ↓
解析引擎:  正则匹配 → 字段提取 → 去重 → 匹配
     ↓
存储层:    SQLite(packages) + SharedPreferences(设置)
     ↓
业务逻辑:  状态迁移 | 提醒调度 | 数据导入导出
     ↓
UI展示:    首页列表 | 代取弹窗 | 详情页 | 设置页
`

### 5.2 核心功能详述

#### SMS自动识别 (P0)
- 启动时扫描收件箱，实时监听新短信
- 13+快递公司正则解析
- 提取: 取件码、地址、电话、截止时间、营业时间
- 三级去重: 内存LRU Set → 数据库查重 → 全局合并
- 验收: 识别准确率>95%, 扫描速度<3秒/100条

#### 购物App通知监控 (P1)
- NotificationListenerService监控14个App
- 提取商品名/单号/公司/地址
- 等待SMS补充取件码
- 验收: 捕获率>90%, 解析准确率>80%

#### 代取分享与弹窗 (P0, v2.0新增)
- 微信通知监控检测代取标记
- 标准化分享文本格式
- App前台时自动弹窗提示
- 接受后自动导入，pushedBy/pushStatus记录
- 首页“代取”标签展示
- 验收: 检测准确率>99%, 弹窗<1秒

#### 包裹管理 (P0)
- 筛选: 待取件/代取/已取件/已过期
- 排序: 最新/最早/按驿站/按截止
- 批量操作: 标记已取/分享/删除
- 取件码横幅: 大号展示, 一键复制/拨号/导航

#### OCR截图识别 (P1)
- Google ML Kit Text Recognition (Chinese)
- 支持单图/多图批量
- 验收: 单号>90%, 公司>85%

## 6. 用户流程: 代取场景

`
推送方(老婆)                  接收方(老公)             取件通
  |                          |                    |
  |包裹详情→“找人代取”        |                    |
  |→微信发送代取文本        |                    |
  |                          |微信通知栏推送     |
  |                          |打开App            |检测代取标记
  |                          |                    |→解析字段
  |                          |                    |→弹窗提示
  |                          |点“接受并导入”   |
  |                          |                    |→建包裹
  |                          |                    |  pushedBy=“老婆”
  |                          |                    |  pushStatus=“received”
`
